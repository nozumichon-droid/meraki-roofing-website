⚠ Critical Issues (Fix Immediately)

Exposed API Keys & Webhook URLs
Your Google Cloud API key and Google Sheets webhook URL are hardcoded in the HTML source of multiple pages. Anyone can view source and abuse these credentials — running up your Google Cloud bill, spamming your lead sheet, or hijacking form submissions.
  • CRITICAL Google API Key exposed in plain text Found in: index.html, estimate.html
    The key AIzaSyAE4... is visible to anyone viewing page source. It enables Maps, Places, and Solar API calls billed to your account.
    Fix: Restrict the API key in Google Cloud Console to your domain only, or move to a server-side proxy.
  • CRITICAL Google Sheets Webhook URL exposed Found in: index.html, estimate.html, partners.html
    Anyone can POST fake leads to your spreadsheet.
    Fix: Add validation/secret token in your Apps Script, or move to a backend handler.
  • CRITICAL Email address exposed in schema markup Found in: index.html (line ~4243) — axmartinez22@gmail.com
    Fix: Use a business email (e.g., info@merakiroofing.com) or remove from schema.

SEO Recommendations

  • HIGH Add LocalBusiness schema to all inner pagesOnly 3 of 18 pages have JSON-LD structured data. Adding RoofingContractor/LocalBusiness schema to every page improves search visibility and rich snippet eligibility.
  • HIGH Add OG images to 6 pages missing themPages without og:image won't generate visual previews when shared on social media — a missed opportunity for residential, gallery, blog, partners, and drone pages.
  • MEDIUM Add Twitter Card meta tagsOnly 2 pages have Twitter-specific meta. Add twitter:card, twitter:title, and twitter:description site-wide.
  • MEDIUM Fix canonical URL on financing.html and drone-inspections.htmlUse standard <link rel="canonical"> format. Drone page includes .html extension in canonical which may cause issues.
  • LOW Add canonical URL to partners.htmlCurrently the only page missing a canonical tag entirely.

Code Performance

  • HIGH Massive inline CSS/JS in homepageindex.html is 9,348 lines with ~200KB of inline CSS and 8,500+ lines of JavaScript. Extract to external files for caching and parallel loading.
  • MEDIUM No lazy loading on imagesBelow-fold images don't use loading="lazy". Adding this attribute would significantly improve initial page load.
  • MEDIUM External API calls on every page loadWeather API (Open-Meteo) and QR code generation (qrserver.com) run on page load. Consider caching weather data and generating QR codes at build time.
  • LOW Three Google Fonts loaded (12 weights total)Lora, Inter, and Playfair Display with 4 weights each. Consider reducing to 2-3 weights per font.

Accessibility Audit

What's Working Well

Good semantic HTML structure with proper <nav>, <main>, <footer>, and <section> tags. Forms have proper labels and ARIA attributes. Skip navigation link is present. Mobile toggle has aria-expanded. Error messages use role="alert".

Issues to Address

  • MEDIUM Color contrast concernTerracotta (#FF5C34) on light backgrounds may fail WCAG AA for normal-size text. Test with a contrast checker and consider darkening to #E5482A for body text.
  • MEDIUM SVG icons missing ARIA labelsDecorative SVGs in nav and content sections lack aria-hidden="true" or descriptive aria-label attributes.
  • LOW Typewriter animation accessibilityThe hero heading uses a typewriter effect that may not convey content to screen readers properly. Ensure the full text is available in the DOM.
  • LOW Map markers lack ARIA labelsLeaflet map pins use custom divs without accessible text alternatives.

Cross-Page Consistency

  • HIGH drone-inspections.html has different navigationUses a simplified nav (Home | Residential | Commercial | Free Estimate | Contact Us) instead of the full nav with dropdown menu, Storm Damage, Maintenance, and phone number. This creates a broken user flow.
  • MEDIUM partners.html dropdown menu order differsItems in the resources dropdown appear in a different sequence than other pages. Should match site-wide order.
  • MEDIUM Broken image referenceblog-insurance-claims-guide.html references images/logos/meraki-icon.svg which doesn't exist.
  • LOW Inconsistent GA trackingSome pages have GA_MEASUREMENT_ID placeholder while others don't include Google Analytics at all. Standardize tracking across all pages.
